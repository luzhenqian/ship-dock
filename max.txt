⏺ Based on the exploration, shipdock.web3noah.com 是要给 Ship Dock
  平台本身配置域名（不是给部署的项目）。这需要在 服务器层面 操作，不是通过 Ship
  Dock 的项目域名管理功能。

  步骤如下：

  1. 配置 DNS 解析

  在你的域名管理商（管理 web3noah.com 的地方）添加一条 A 记录：

  ┌──────┬──────────┬─────────────────┐
  │ 类型 │ 主机记录 │       值        │
  ├──────┼──────────┼─────────────────┤
  │ A    │ shipdock │ 178.104.154.165 │
  └──────┴──────────┴─────────────────┘

  如果你已经在 Ship Dock 的 Domains 页面配置了 DNS
  provider（Namecheap/GoDaddy），你可以直接在 Ship Dock 的 /domains
  页面操作添加这条记录。

  2. 在服务器上配置 Nginx

  SSH 到服务器，创建 Nginx 配置：

  sudo nano /etc/nginx/sites-available/shipdock.conf

  内容大致为（先配 HTTP，获取证书后再加 HTTPS）：

  server {
      listen 80;
      server_name shipdock.web3noah.com;

      location / {
          proxy_pass http://127.0.0.1:<前端端口>;
          proxy_set_header Host $host;
          proxy_set_header X-Real-IP $remote_addr;
          proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
          proxy_set_header X-Forwarded-Proto $scheme;
      }
  }

  然后启用并重载：

  sudo ln -s /etc/nginx/sites-available/shipdock.conf /etc/nginx/sites-enabled/
  sudo nginx -t && sudo systemctl reload nginx

  3. 配置 SSL 证书

  sudo certbot --nginx -d shipdock.web3noah.com

  Certbot 会自动修改 Nginx 配置添加 HTTPS。

  ---
  关键问题：你需要确认前端在服务器上跑在哪个端口。你能看一下服务器上 pm2 list
  的输出吗？可以在终端运行 ! ssh <你的服务器> pm2 list 来查看。
